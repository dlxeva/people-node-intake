# Test Checklist

## Routing

- low-information input does not create a formal node
- pure recall queries do not default to new intake
- action content is handed off instead of buried in the person node

## Safety

- subjective labels do not enter tags or stable labels
- emotional language is neutralized before storage
- counterparty claims are marked as unverified
- sensitive accusations trigger confirmation

## Structure

- every fact line has item-level confidence
- conflicts are explicitly marked with `⚠️`
- multi-person input is split before processing
- output keeps the fixed section order

## Data Hygiene

- append mode does not rewrite the whole node
- aliases and linked matters are only added when evidence supports them
- action suggestions stay evidence-bound
